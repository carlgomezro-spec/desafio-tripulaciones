Module: models/adminModel

Modelo que maneja todas las operaciones CRUD y consultas administrativas relacionadas con ventas, clientes, productos, RRHH y usuarios. Incluye validaciones de datos y manejo de contraseñas con bcrypt.

(async, static) updateUserById(user_id, userData) → {Promise.<Object>}

Actualización parcial: solo los campos proporcionados se actualizan. Contraseña: si se proporciona string vacío (""), se mantiene la contraseña actual. Si se proporciona nueva contraseña, se hashea automáticamente.
Parameters:
Name Type Description
user_id number ID del usuario a actualizar
userData Object Campos a actualizar (todos opcionales)
Properties
Name Type Attributes Description
role string <optional>
 Nuevo rol del usuario
email string <optional>
 Nuevo email del usuario
password string <optional>
 Nueva contraseña (cadena vacía mantiene la actual)
Source:
  • models/adminModel.js, line 199
Throws:
Usuario no encontrado, rol inválido o error de base de datos
Type
Error
Returns:
Objeto con mensaje y datos del usuario actualizado
Type
Promise.<Object>
Examples
// Actualizar solo el rol:
await updateUserById(123, { role: 'hr' });
// Actualizar email y contraseña:
await updateUserById(123, { 
  email: 'nuevo@email.com',
  password: 'NuevaPass123' 
});
// Mantener contraseña actual:
await updateUserById(123, { 
  email: 'nuevo@email.com',
  password: ""  // ← cadena vacía = mantener contraseña
});
